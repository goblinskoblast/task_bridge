# Отчет по изменениям за 7 апреля 2026

## Что сделано сегодня

### 1. Усилена безопасность и внутренняя связка сервисов
- `data_agent` закрыт внутренним `X-Internal-Token`.
- Внутренние вызовы между `web` и `task_bridge` переведены на единый защищенный механизм.
- Добавлен fallback derivation для `WEBHOOK_URL` и `INTERNAL_API_TOKEN`, чтобы Railway-переменные не ломали запуск при кривой конфигурации.

### 2. Бот переведен на стабильный webhook lifecycle
- `web` переведен на нормальный webhook runtime без polling-конфликта.
- Убраны deprecated `FastAPI on_event` hooks, lifecycle переведен на `lifespan`.
- Railway-логи после redeploy больше не показывают `DeprecationWarning` для webhook startup/shutdown.
- В свежих логах не наблюдается новый `TelegramConflictError`.

### 3. Улучшена диагностика `data_agent`
- Добавлены `trace_id`, `debug_summary`, `debug_payload`, `scenario`, `status`.
- В Telegram работает `/agentdebug`.
- При падении monitor flow и report flow пользователь получает более понятную диагностику.
- `/debug` теперь показывает свежий trace, а не устаревший snapshot.

### 4. Улучшена доставка отчетов и мониторингов в Telegram
- Работает выбор отдельного Telegram-чата для отчетов.
- Добавлены управляемые report-flows и monitor-flows для:
  - `reviews`
  - `stoplist`
  - `blanks`
- Добавлены команды управления мониторингом и уведомления о сбоях мониторинга.

### 5. Усилен `Playwright` и browser/data-agent слой
- Улучшено распознавание ошибок `login / 2FA / captcha / access denied`.
- В diagnostics теперь есть `stage`, `url`, `last_action` и техническая причина.
- `stoplist` и `blanks` теперь не маскируют часть проблем под ложный успешный результат.

### 6. Исправлены реальные баги, найденные live-smoke на Railway
- Исправлен routing `data_agent`:
  - запросы про `бланки` теперь действительно идут в `blanks_report`
  - `reviews` без точки больше не подхватывают старую точку из прошлой сессии
- Исправлен carryover слотов между несвязанными сценариями.
- Улучшена фильтрация `stoplist`, чтобы не тащить в список цены, кнопки действий и мусорные строки.
- `reviews_report` без `REVIEWS_SHEET_URL` теперь маркируется как `failed`, а не как `completed`.

## Текущее реальное состояние проекта

Проект стал заметно стабильнее на уровне архитектуры, webhook, внутренних API и диагностируемости.

Главный прогресс дня:
- теперь проще понять, почему именно упал `data_agent`
- Railway-логи уже можно использовать как нормальный рабочий инструмент
- критичные маршруты `reviews / blanks / stoplist` начали вести себя предсказуемо по статусам и trace

Главные оставшиеся live-проблемы:
- не настроен `REVIEWS_SHEET_URL`, поэтому reviews-report пока не работает как боевой источник
- на живых страницах Italian Pizza все еще недостаточно надежен выбор точки:
  - в публичном `stoplist`
  - в портале `blanks`
