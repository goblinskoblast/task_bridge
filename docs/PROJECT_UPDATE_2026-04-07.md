# Отчет по изменениям за 7 апреля 2026

## Что сделано сегодня

### 1. Telegram UX для агента приведен в безопасное состояние
- Убраны автоматические diagnostic-блоки из обычных ответов Telegram-бота.
- Убраны пользовательские упоминания `/agentdebug` из приветствия и кнопок агента.
- Для обычного пользователя `/agentdebug` теперь недоступен.
- Доступ к `/agentdebug` сохранен только для `DEVELOPER_TELEGRAM_ID`.
- Для длинных agent-запросов сохранен быстрый webhook-ack и фоновая обработка, но стартовое сообщение `Принял запрос...` временно отключено.
- Ошибки по `stoplist`, `blanks`, `reviews` в Telegram переведены на короткие нейтральные шаблоны без trace/stage/url.

### 2. Мониторинги тоже очищены от технического шума
- Ошибки мониторингов больше не отправляют в Telegram подробные diagnostic summary.
- Для `stoplist`, `blanks`, `reviews` мониторинг теперь шлет только короткое нейтральное сообщение об ошибке.
- При этом внутренняя диагностика и payload по мониторингам сохраняются в системе для отладки.

### 3. `stoplist` переведен на быстрый и более надежный SSR-путь
- Для публичного сайта Italian Pizza добавлен fast-path без зависимости от Playwright UI.
- Адаптер теперь сначала читает HTML страницы точки напрямую.
- Подтверждение точки выполняется по поддомену и SSR-данным страницы.
- Недоступные позиции извлекаются из SSR-данных по `status="stop_list"`.
- Старый browser-flow оставлен как fallback, если fast-path не сработал.
- Исправлен ложный `403` в детекторе публичных ошибок: CSS-значения страницы больше не принимаются за access denied.

### 4. `blanks` усилены по детерминированным шагам
- После логина теперь отдельно проверяется, не осталась ли открытой форма авторизации.
- Если период запрошен, но не выбран, сценарий завершается как `failed`, а не идет дальше с ложным `ok`.
- Перед чтением отчета теперь отдельно проверяется, что портал действительно открыл экран бланков/перегрузок, а не меню или страницу входа.
- Нормализация текста бланков очищена от навигационного мусора: `главная`, `настройки`, `профиль`, `выйти`, `логин`, `пароль` и похожие строки больше не должны попадать в пользовательский отчет.

### 5. `reviews` переведены на sheet-first по точке
- Для запросов по точке агент теперь сначала использует `REVIEWS_SHEET_URL`, если он настроен.
- Browser/maps-path для отзывов используется только если пользователь явно просит `2GIS`, `Яндекс`, `карты`, `maps`, либо если sheet не настроен.
- В `review_report` добавлен point-filter по branch/location/store/point полям.
- Поддержаны алиасы точек через нормализацию названий и итальянские точки проекта.
- Состояние `нет данных` отделено от технической ошибки: пустая выборка по точке или периоду теперь остается `ok`, а не выглядит как поломка.

## Текущее состояние проекта после этого пакета

### Уже стало лучше
- Обычный пользователь больше не видит технические trace/debug-сообщения от агента.
- Долгие запросы больше не должны провоцировать повторный пользовательский спам из-за подвисшего webhook.
- `stoplist` стал заметно надежнее и быстрее за счет SSR-пути.
- `reviews` по точке архитектурно переведены на более предсказуемый источник данных.

- Нужно прогнать этот пакет уже на Railway после deploy и проверить фактическое поведение `web` и `task_bridge`.
