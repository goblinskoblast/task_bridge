# SDD в TaskBridge

Этот каталог вводит Spec-Driven Development (SDD) для изменений в проекте.

## Цели
- Уменьшить регрессии в `bot/`, `webapp/`, `db/`.
- Фиксировать требования до реализации.
- Привязывать реализацию к проверяемым acceptance criteria.

## Структура
- `process.md` — обязательный процесс SDD.
- `templates/` — шаблоны для спека и task-листа.
- `specs/` — утвержденные спецификации.
- `tasks/` — декомпозиция и чек-листы выполнения.
- `adrs/` — архитектурные решения.

## Базовое правило
Любое существенное изменение в `bot/`, `webapp/`, `db/` начинается со спека в `docs/sdd/specs/`.
## Как это применено к OpenClaw агенту
- В коде `bot/openclaw_client.py` включено автоматическое добавление system-инструкции со спецификацией в каждый запрос (`/v1/responses`).
- Источник спецификации задается через `OPENCLAW_SDD_SPEC_PATH`.
- Включение/выключение: `OPENCLAW_ENFORCE_SDD_SPEC=True|False`.
- Ограничение размера загружаемого текста спека: `OPENCLAW_SDD_MAX_CHARS`.

По умолчанию enforcement включен и использует:
`docs/sdd/specs/SPEC-OC-001-openclaw-agent.md`
